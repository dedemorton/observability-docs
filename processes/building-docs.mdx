## Old system

* We use asciidoc to write our documentation and asciidoctor to build it. A guide for formatting and building our documentation is available in the [docs readme](https://github.com/elastic/docs/blob/master/README.asciidoc).
* Shorthand [build aliases](https://github.com/elastic/docs/blob/master/doc_build_aliases.sh) are also available (and quite useful).
    * Append the `--open` flag to spin up a local server and view the changes you've made.
