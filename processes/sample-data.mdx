## Sample data


### APM integration testing

To add logs and metrics to an ESS instance, you can use [`apm-integration-testing`](https://github.com/elastic/apm-integration-testing).
Grab the APM Server URL, Secret token, and Elasticsearch URL from the Cloud console,
and export them as environment variables:

```bash
export APM_SERVER_URL=https://dddfafafaf.apm.us-east4.gcp.elastic-cloud.com \
export APM_SERVER_TOKEN=QTEEfSkVOFUddd5C5aJJB \
export ES_URL=https://dddfafafaf.es.us-east4.gcp.elastic-cloud.com:9243
```

Then, spin up Opbeans at `localhost:3000` watch the data flow into ESS:

```
./scripts/compose.py start 8.0 --no-elasticsearch --no-kibana --no-apm-server \
--with-opbeans-go --with-opbeans-java --with-opbeans-node --apm-server-url=${APM_SERVER_URL} \
--apm-server-secret-token=${APM_SERVER_TOKEN} --opbeans-elasticsearch-url=${ES_URL} --with-elastic-agent
```

NOTE: Adding a local Elastic Agent via this method is currently broken.

### Connecting to the Observability development clusters

Read more about these clusters [here](https://github.com/elastic/observability-dev/blob/main/docs/observability-clusters.md).

NOTE: This method is potentially being deprecated

. Check out the version of Kibana that you want to spin up.
. Get the [credentials](https://github.com/elastic/observability-dev/blob/main/docs/observability-clusters.md) of the matching Observability cluster. The versions must match.
. Create a `kibana.dev.yml` file in the `config` directory and add the following:
+
```yml
elasticsearch.hosts: REPLACE_ME
elasticsearch.username: kibana_system_user
elasticsearch.password: REPLACE_ME
elasticsearch.ssl.verificationMode: none
```

. Run `nvm use`, `yarn kbn boostrap`, and `yarn start`.
