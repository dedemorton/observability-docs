[[syslog-input]]
== Syslog input

Use the `syslog` input to read events over TCP, UDP, or a Unix stream socket.
This input will parse BSD (rfc3164) events and some variants.

Example configurations:

[source,yaml]
----
inputs:
  - type: syslog
    protocol.udp:
      host: "localhost:9000"
----

[source,yaml]
----
inputs:
  - type: syslog
    protocol.tcp:
      host: "localhost:9000"
----

[source,yaml]
----
inputs:
  - type: syslog
    protocol.unix:
      path: "/path/to/syslog.sock"
----

The `syslog` input supports the following settings, grouped by category.
Many of these settings have sensible defaults that allow you to run {agent}
with minimal configuration.

<<input-syslog-udp-settings>>::
Settings for reading over UDP.

<<input-syslog-tcp-settings>>::
Settings for reading over TCP.

<<input-syslog-unix-settings>>::
Settings for reading over a Unix socket.

<<input-syslog-data-parsing-settings>>::
Settings used to parse, filter, and transform data.
